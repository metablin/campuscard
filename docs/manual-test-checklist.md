# Ручной чеклист CampusCard

Прогон перед демо/релизом. Окружение: backend `uvicorn app.main:app --port 8000`,
frontend `npm run dev` (http://localhost:5173) **или** `docker compose up`
(http://localhost). База наполнена: `python -m app.seed`.

Легенда: ☐ не проверено · ☑ ок · ✗ баг (завести issue).

## Вход / выход

- [ ] **1. Dev-вход.** Открыть `/login` → «Войти как демо-пользователь» →
  редирект на `/app`, отображается имя пользователя. В DevTools → Application →
  Cookies: кука `campuscard_session` с флагом HttpOnly.
- [ ] **2. Защита роутов.** Без входа открыть `/app` и `/edit` напрямую —
  редирект на `/login`.
- [ ] **3. Сессия переживает перезагрузку.** Обновить страницу `/app` —
  пользователь остаётся залогинен (GET /api/auth/me возвращает 200).
- [ ] **4. Выход.** Нажать «Выйти» → редирект на `/login`, кука удалена,
  `/app` снова недоступен.
- [ ] **5. Кнопка VK ID.** При пустом `VITE_VK_APP_ID` кнопка «Войти через
  VK ID» скрыта (локально проверяется только dev-вход; VK ID — на порту 80
  или через внешний домен, см. AGENTS.md).

## Создание и редактирование визитки

- [ ] **6. Создание.** «Создать визитку» → заполнить ФИО, вуз, slug
  (или «Сгенерировать» из ФИО — кириллица транслитерируется) → «Сохранить» →
  редирект на `/app`, визитка отображается.
- [ ] **7. Валидация slug.** Ввести `Ab`, `ab!`, `a` (2 символа), 41 символ,
  `admin` — кнопка «Сохранить» не даёт сохранить, показаны понятные ошибки.
- [ ] **8. Занятый slug.** Попробовать slug `anna-smirnova` (занят seed-визиткой) —
  индикатор «занято», при сохранении ошибка 409 с текстом на русском.
- [ ] **9. Границы полей.** Описание ровно 1000 символов сохраняется;
  1001 — ошибка. 15 навыков — ок, 16-й не добавляется. 10 ссылок — ок.
- [ ] **10. Ссылки.** Добавить ссылки vk/telegram/email/phone/github/site.
  Ссылка со схемой `javascript:alert(1)` отклоняется валидацией.
- [ ] **11. Редактирование.** Изменить ФИО и тему → «Сохранить» → изменения
  видны на `/app` и на публичной странице.
- [ ] **12. Смена slug.** Поменять slug → старая публичная ссылка даёт 404,
  новая работает.

## Публикация и публичная страница

- [ ] **13. Переключатель публикации.** Тумблер на `/app`: «Визитка скрыта» →
  «Визитка видна всем», счётчик просмотров отображается.
- [ ] **14. Публичная страница.** Открыть `/u/<slug>` — имя, вуз, навыки,
  ссылки, выбранная тема оформления. В HTML/JSON нет приватных полей
  (user_id, email, vk_id) — проверить ответ `GET /api/u/<slug>` в DevTools.
- [ ] **15. Скрытая визитка → 404.** Снять с публикации → `/u/<slug>` отдаёт
  страницу «не найдено» (и API 404) — как для несуществующей, без подсказок.
- [ ] **16. Приватное окно.** Открыть опубликованную `/u/<slug>` в окне
  инкогнито (без куки) — визитка доступна без входа.
- [ ] **17. QR-код.** На `/app` отображается QR → «Скачать QR (PNG)»
  скачивает файл. Отсканировать телефоном (или декодером) — URL ведёт на
  `/u/<slug>` и страница открывается. На публичной странице «Сохранить
  контакт» скачивает vCard (.vcf), контакт импортируется в телефон.
- [ ] **18. Счётчик просмотров.** Обновить публичную страницу 2–3 раза →
  счётчик «просмотров» на `/app` увеличился.

## Удаление и устойчивость

- [ ] **19. Удаление визитки.** «Удалить визитку» → подтверждение в ActionSheet
  → визитка удалена, `/app` показывает состояние «нет визитки»,
  `/u/<slug>` → 404. После этого можно создать визитку заново.
- [ ] **20. Мобильная вёрстка.** DevTools → device toolbar (iPhone/Android,
  360–390px): лендинг, `/login`, форма редактирования, публичная визитка —
  без горизонтального скролла, кнопки не обрезаны, QR читаем.

## Смоук-тест Docker (опционально)

- [ ] **21.** `docker compose up -d` → http://localhost открывается,
  `/api/health` = ok, seed-визитка `/u/anna-smirnova` доступна
  (см. README, раздел «Запуск в Docker»).
