# Дизайн-устав CampusCard

Правила оформления всех экранов. Цель — единообразный, предсказуемый UI
в рамках дизайн-системы VK. Стиль VK (цвета, токены, компоненты) не меняем.

## 1. Только VKUI
- Все визуальные элементы — компоненты и токены `@vkontakte/vkui`
  (установленной версии) и иконки `@vkontakte/icons`.
- Самописный CSS (inline-стили) допустим **только для раскладки**:
  `display: flex/grid`, `justifyContent`, `alignItems`, `gap`,
  `maxWidth`, `width`, `margin: 0 auto`.
- Запрещено переопределять цвета, тени, радиусы, шрифты и отступы,
  которые задаёт VKUI. Исключение — тематическая шапка публичной визитки,
  цвета которой берутся из `@vkontakte/vkui-tokens` (см. `edit/themes.ts`).

## 2. Иерархия действий
- На каждом экране — **ровно одна primary-кнопка** (основное действие).
- Вторичные действия — `mode="secondary"` или `mode="tertiary"`.
- Опасные действия (удаление) — `appearance="negative"` / destructive,
  визуально отделены от primary (отдельная группа/секция), с подтверждением
  через `Alert`.

## 3. Вертикальный ритм
- Отступы между блоками — только через `Group`, `Card`, `Div`, `Spacing`,
  `Separator` из VKUI.
- Никаких «магических» пиксельных отступов в стилях
  (`paddingTop: 48` и т.п.). Ритм задают компоненты VKUI.

## 4. Адаптивность
- Mobile-first: минимальная ширина 360px, контент не ломается.
- На десктопе контент центрируется с `max-width`:
  - формы/дашборд — 640px,
  - публичная визитка — карточка по центру (480px).
- Центрирование — через обёртку (`PageContainer`) или `SplitLayout`/`SplitCol`;
  раскладка только flex/grid, без фиксированных пиксельных ширин у текста.

## 5. Состояния экранов
У каждого экрана три состояния:
- **loading** — `PanelSpinner` (или `ScreenSpinner` для модальных операций);
- **empty** — `Placeholder` с понятным текстом и primary-действием;
- **error** — `Placeholder` с текстом ошибки и кнопкой «Повторить» /
  возвратом на безопасный экран.

## 6. Формы
- Каждое поле — в `FormItem` с подписью (`top`) и `status="error"` при ошибке;
  текст ошибки/подсказка — в `bottom` или `FormStatus`.
- Кнопка отправки — `stretched`, внизу формы, одна (primary).
- Валидация — до отправки (совпадает с бэкендом), ошибки собираются списком.

## 7. Публичная визитка
- Карточка по центру экрана, шапка в цветах темы визитки
  (токены `@vkontakte/vkui-tokens`), остальное — стандартные `Group`/`CellButton`.
- Кнопка «Сохранить контакт» — единственная primary на странице.
- Footer со ссылкой «Создай свою визитку на CampusCard» → `/`.
